# ADR-009 테스트·품질 전략 — "시연을 깨뜨릴 수 있는 것만 테스트한다"

## 상태
승인 (2026-07-20, 결정자: 이윤서)

## 맥락
- 2인 3주, D15 공개 URL E2E 시연이 유일 성공 조건. 테스트는 시연 리스크 감소 수단으로만 정당화.

## 검토한 선택지
- 커버리지 목표 설정(반려 — 수치 채우기 노동), FE 컴포넌트/스냅샷 테스트(반려 — 가치 최저), CI 테스트 게이트(반려 — ADR-007 정합)

## 결정
1. **pipeline (pytest 2종)**
   - D8 유틸 단위 테스트 — 함정 재현 케이스 그대로(cp949 파일, NFD 문자열, 스왑 값, 비패딩 월).
   - seed 산출 검증 — 스키마 컬럼·NOT NULL·score 내림차순=order_key 정합·탐사쿼터 5~10%·region_type 분포 sanity·시연 시군구 행수 범위.
2. **BE**: 통합 테스트만 — 7 엔드포인트 happy path + 엣지 3종(중복 회신, 빈 큐 발행, 역할 외 접근). Testcontainers로 실제 Postgres 대상.
3. **FE**: 자동 테스트 0.
4. **E2E**: Playwright 스모크 1본 — 데모 시나리오 그대로(로그인→관제 큐→발행→모바일 회신→관제 반영·카운터 증가). D13부터 배포 URL 대상 실행.
5. **수동 QA**: D12 체크리스트(빈값/로딩/오류 상태, 모바일 뷰포트) — WBS 유지.

## 근거
- E2E 스모크 = 시연 리허설의 자동화 — D14~15 리허설 비용 절감 겸용.
- BE는 CRUD 위임 구조라 서비스 단위 테스트의 정보 가치가 없음.

## 결과 (트레이드오프 수용)
- FE 리그레션은 수동 QA에 의존 — 화면 2계통 규모라 수용.
- Testcontainers로 로컬 테스트에 Docker 필요(ADR-003과 동일 의존이라 추가 비용 0).
